# spec-2 — 손님② 수입사 한정주 이미지 페이지 (자유 배치 스티커)

> 출처 시안: `design-reference/page-2.html` (768×1024 세로, 태블릿 페이지형 디지털 메뉴판 한 페이지)
> 대상 구현: Next.js + React + CSS Modules / CSS 변수, 반응형 실제 DOM

---

## 1. 목적 / 역할

수입사가 제공한 **한정주·이벤트 비주얼 이미지를 전면(full-bleed) 배경으로 보여주고**, 그 위에 매장이 자유롭게 위치를 잡은 **회전 스티커(라벨·뱃지·가격 카드)** 를 얹는 "이미지형 메뉴 페이지". 텍스트 목록(page-1)과 달리, 정형 그리드가 아니라 **이미지 + 자유 배치 오버레이** 가 핵심인 슬라이드형 페이지다.

---

## 2. 전체 레이아웃 구조

세로 3분할 + 절대 배치 오버레이 레이어 구조. 좌표가 아니라 흐름/정렬로 기술한다.

```
┌───────────────────────────────────────────┐
│ ① 헤더 (고정 높이, 다크바)                  │  height ≈ 60px 비율
├───────────────────────────────────────────┤
│ ② 본문 = 이미지 캔버스 (flex:1, 가변)       │
│   - 좌우/상하 동일 거터(gutter)             │
│   - 점선 테두리 placeholder (이미지 미등록)  │
│   - 중앙: 이미지 placeholder 안내 (빈 상태)  │
│   - 오버레이 레이어: 자유 배치 스티커들      │
│   - 캔버스 좌/우 바깥쪽: ‹ › 페이지 화살표   │
├───────────────────────────────────────────┤
│ ③ 푸터 (고정 높이): dot 인디케이터 + 카운터  │  height ≈ 78px 비율
└───────────────────────────────────────────┘
```

- **루트 컨테이너**: 페이지 비율 768:1024 (3:4)를 유지하는 세로 프레임. 태블릿 가로 폭에 맞춰 스케일, 데스크톱에서는 중앙 정렬 + 최대폭 제한. 배경 `#FDFCFB`(시안 캔버스), 프레임 바깥 여백 배경은 `#E7E2D7`.
- **헤더**: 폭 100%, 다크바. `display:flex; align-items:center`. 좌측 = 로고 묶음(골드 사각 아이콘 + "THE HAND" + 서브타이틀), 우측 끝 = 섹션 태그("한정주"). 좌우 패딩 ≈ 28px(시안 기준).
- **본문 캔버스**: 헤더 아래 ~18px 갭 뒤 시작, 좌우 거터 24px, 라운드 14px의 점선 박스. 내부는 `position:relative` 컨테이너이며 모든 스티커는 이 컨테이너 기준 절대 배치. 이미지가 등록되면 이 박스가 곧 이미지 컨테이너(`object-fit:cover`)가 되고 점선/빈 상태 안내는 사라진다.
- **본문 정렬**: 빈 상태의 안내 아이콘·문구는 캔버스 **정중앙 가로 center** 정렬, 수직으로도 거의 중앙(시안에서 y≈442~558 영역).
- **푸터**: 폭 100%, 상단에 얇은 구분선. 가운데에 dot 인디케이터 묶음(가로 center 정렬), 좌측 끝에 "← 좌우로 넘기기" 힌트, 우측 끝에 "N / 10" 카운터.
- **오버레이 화살표**: 캔버스 수직 중앙 높이에 좌(‹)·우(›) 큰 글리프, 화면 좌우 가장자리에 위치. `opacity 0.5`로 은은하게.

## 3. 컴포넌트 명세

### 3.1 헤더 바 `<MenuHeader>`
- **형태/치수**: 가로 full-bleed, 높이 60px(비율). 배경 `#111418`.
- **로고 아이콘**: 28×28, `border-radius:7px`, 골드 그라데이션 채움 — `linear-gradient(135deg, #f0c97a → #c0792a)`. 중앙에 한자 "手", Noto Serif KR 700, 15px, 색 `#1e1e36`.
- **워드마크**: "THE HAND" — Pretendard 800, 14px, 색 `#F3EDE2`, letter-spacing `0.14em`.
- **서브타이틀**: "사케 바 · 디지털 메뉴판" — Pretendard 400, 10px, 색 `#B8AE9C`. 워드마크 바로 아래.
- **섹션 태그(우측)**: "한정주" — Pretendard 700, 12px, 색 `#C9A24B`(골드), letter-spacing `0.1em`, 우측 정렬. (page-1은 "日本酒" → 페이지별로 텍스트만 교체되는 슬롯)

### 3.2 이미지 캔버스 / placeholder `<ImageCanvas>`
- **형태/치수**: 캔버스 영역 720×856(비율), `border-radius:14px`.
- **빈 상태 박스**: 채움 `#EDE7DD`, 테두리 `1.5px dashed #D8D0C4`(dash 패턴 6 5).
- **빈 상태 안내 아이콘**: 중앙 "사진" 글리프 — 모서리 둥근 사각 프레임(88×68, r8) + 좌상단 원(해) + 산 모양 폴리라인. 선색 `#A99E8C`, `stroke-width:2`, 전체 `opacity:0.5`.
- **빈 상태 안내 문구(2줄, 중앙)**:
  - 1줄: "수입사 제공 이미지 영역" — Pretendard 700, 14px, 색 `#8E8576`.
  - 2줄: "관리 화면에서 한정주·이벤트 이미지를 올립니다" — Pretendard 400, 11.5px, 색 `#A99E8C`.
- **이미지 등록 상태(추론)**: 점선·빈 상태 UI를 숨기고 등록 이미지를 `object-fit:cover`로 캔버스 가득 채움. 라운드 14px 유지, 스티커 오버레이는 그대로 위에 표시.

### 3.3 자유 배치 스티커 (공통) `<Sticker>`
- 모든 스티커는 캔버스 기준 **절대 위치 + 개별 회전(rotate)** 을 가지는 오버레이.
- 공통 그림자 두 종류:
  - `--shadow-sticker`(sh): `0 3px 4px rgba(26,26,46,0.28)` (filter는 stdDeviation 4 → blur≈4).
  - `--shadow-sticker-lg`(sh2): `0 5px 7px rgba(26,26,46,0.22)` (blur≈7, 더 큰 카드용).
- 회전값은 데이터로 들고 `transform: rotate(deg)`. 시안 실측 회전·위치는 4종 모두 제각각(아래 개별 명세). 이 값들은 **기본 시드값**이며 라이브에서는 관리자가 드래그/회전으로 갱신.

#### 3.3a 리본 라벨 스티커 (좌상단) — `<RibbonSticker>`
- **형태**: 좌측은 직사각형, 우측 끝이 **삼각 꼬리(화살 깃발)** 모양. 본체 약 97×28, r4. 꼬리는 본체 우측에서 +11px 돌출한 삼각형.
- **색**: 채움 `#1F8A5B`(녹색). 텍스트 흰색.
- **텍스트**: "계절 한정주" — Pretendard 800, 13px, `#fff`, 중앙 정렬.
- **상태/회전**: 시드 `rotate(-7deg)`, 좌상단 배치.

#### 3.3b 원형 카운트 뱃지 (우상단) — `<CircleBadge>`
- **형태**: 지름 52px 원(r26). 채움 `#C84B31`(레드/주황). 그림자 sh.
- **텍스트(2줄, 중앙)**:
  - 1줄: "한정" — Pretendard 800, 11px, `#fff`.
  - 2줄: "30병" — Pretendard 700, 9px, `#fff`.
- **상태/회전**: 시드 `rotate(8deg)`, 우상단 배치. (수량은 데이터 → "한정 N병")

#### 3.3c 캡슐 칩 스티커 — `<PillSticker>`
- **형태**: 알약형 캡슐 ~92.5×27, `border-radius:13.5px`(완전 라운드). 채움 `#fff`, 테두리 `1.5px solid #D8D0C4`.
- **텍스트**: "잔술만 가능" — Pretendard 800, 12.5px, 색 `#2D3436`, 중앙.
- **상태/회전**: 시드 `rotate(6deg)`. 그림자 sh.
- **변형(추론)**: 동일 형태로 "도쿠리·잔술만", "보틀만" 등 판매단위 제한 라벨 재사용 가능.

#### 3.3d 가격 정보 카드 스티커 — `<PriceCardSticker>`
- **형태**: 흰 카드 210×82, `border-radius:10px`, 채움 `#fff`, 그림자 sh2(큰 그림자). 내부 패딩 ≈ 16px.
- **타이틀**: "쿠보타 만쥬 (니가타)" — Noto Serif KR 700, 15px, 색 `#2D3436`, 좌측 정렬. (제품명 + 지역)
- **가격 라인(좌=라벨, 우=가격, 2줄)** — 라벨 Pretendard 500 12px `#636E72`(좌), 가격 Pretendard 800 13.5px `#2D3436`(우 정렬):
  - "잔술 100㎖" ⟷ "24,000"
  - "보틀 720㎖" ⟷ "280,000"
- **상태/회전**: 시드 `rotate(-3deg)`. (판매단위 라인은 데이터 배열 → 잔술/도쿠리/보틀 가변)

### 3.4 페이지 이동 화살표 `<PageArrow>`
- **형태**: 텍스트 글리프 ‹ / › — Pretendard 300(얇게), 30px, 색 `#D8D0C4`, `opacity:0.5`.
- **위치**: 캔버스 수직 중앙, 좌·우 화면 가장자리(좌 x≈20, 우 x≈748). 좌=이전, 우=다음.
- **상태(추론)**: 첫/마지막 페이지에서 비활성(흐림/숨김), 호버/탭 시 강조.

### 3.5 푸터 dot 인디케이터 `<PageIndicator>`
- **구분선**: 푸터 상단 가로선, `1px #EDE8E0`(좌우 28px 인셋).
- **인디케이터 묶음**: 가로 center 정렬. 총 10개 페이지 마커가 **세 가지 형태**로 혼재(시안 실측):
  - **원형 dot**(일반 페이지): r3.5 원, 채움 `#D8D0C4`.
  - **현재 페이지 pill**(active): 가로로 늘어난 알약 18×7, r3.5, 채움 `#F39C12`(주황). — 4번째 위치에 있음 = 현재 페이지.
  - **사각 마커**(특수/이미지 페이지 표식, 추론): 7×7, r1.5, 채움 없음 + `1.4px stroke #D8D0C4`. (시안에서 8번째·일부 위치에 사각형이 섞여 있음 → "이미지 슬라이드" 같은 페이지 유형 구분으로 추정)
  - 마커 간격 ≈ 20px.
- **좌측 힌트**: "← 좌우로 넘기기" — Pretendard 500, 11px, 색 `#9A8F80`, 좌측 정렬.
- **우측 카운터**: "4 / 10" — Pretendard 700, 12px, 색 `#9A8F80`, 우측 정렬. (현재/전체)

### 3.6 (참고) 관리자 패널 `<AdminPanel>`
- 이 페이지 시안에는 **관리자 패널 UI 자체는 렌더되지 않음**. 단, 빈 상태 문구("관리 화면에서 … 이미지를 올립니다")가 관리 화면의 존재를 명시. 구현 시 별도 어드민에서 ① 배경 이미지 업로드, ② 스티커 추가/삭제/텍스트편집/색상/위치·회전 드래그를 제공하는 것으로 가정.

## 5. 추론된 실제 인터랙션 (라이브 앱 동작)

손님(뷰어) 모드:
1. **좌우 스와이프 / 화살표 탭으로 페이지 넘기기** — 이 페이지는 10페이지 캐러셀(슬라이드)의 4번째. ‹ › 탭 또는 가로 스와이프로 이전/다음 페이지 이동. dot 인디케이터·"4 / 10" 카운터가 현재 위치 반영.
2. **dot 인디케이터 탭** — 특정 페이지로 점프(또는 단순 표시용일 수 있음; 최소한 active 상태 동기화).
3. **이미지/스티커는 정적 표시** — 손님 모드에서는 탭으로 확대(라이트박스) 정도가 합리적이며, 드래그/편집은 불가.
4. **자동 슬라이드(추론)** — 무인 메뉴판 특성상 일정 시간 무조작 시 자동 넘김(옵션) 가능. 명시 근거는 없음.

관리자(편집) 모드 — 빈 상태 문구로 강하게 시사됨:
5. **배경 이미지 업로드/교체** — 점선 placeholder를 탭해 수입사 제공 이미지를 올림.
6. **스티커 자유 배치** — 스티커를 드래그로 이동, 핸들로 회전(시안의 -7/8/6/-3deg 같은 기울기), 추가/삭제.
7. **스티커 텍스트/색 편집** — 리본·캡슐·원형 뱃지 문구, 한정 수량(N병), 가격 카드의 제품명·판매단위·가격 라인 편집.
8. **판매단위 라인 가변** — 가격 카드에 잔술/도쿠리/보틀 라인을 0~3개 토글하여 표시.

## 6. 구현 주의점 · 모호한 부분

- **자유 배치 = 데이터 기반 절대 좌표**: 스티커 위치/회전은 SVG 픽셀(768×1024)을 그대로 박지 말고, 캔버스 대비 **백분율(%) 좌표 + rotate(deg)** 로 저장해야 반응형에서 비율 유지됨. 시안 값은 시드일 뿐.
- **반응형 스케일 전략**: 768:1024 비율 프레임을 유지하되, 폰트·간격을 px 고정으로 두면 작은 화면에서 깨짐. 컨테이너 쿼리 또는 루트 폰트 스케일(`clamp`)로 정의 권장. 캔버스를 기준으로 한 상대 단위(%, cqw) 사용.
- **빈 상태 vs 이미지 상태 분기**: placeholder(점선+안내문)는 이미지 미등록 시에만. 이미지 등록 시 점선 제거 + cover. 두 상태를 같은 `<ImageCanvas>` 컴포넌트의 prop로 분기.
- **스티커 가독성**: 실제 배경 이미지 위에 흰/녹/빨 스티커가 얹히므로, 이미지 명암에 따라 대비가 약해질 수 있음. 스티커 그림자(sh/sh2)는 가독성 확보용 → 반드시 유지.
- **dot 인디케이터 형태 혼재의 의미 불명확**: 원 dot / 주황 pill(현재) / 빈 사각형이 섞여 있음. pill이 "현재 페이지"인 것은 명확하나, **사각 마커의 의미가 모호**(이미지 페이지 유형 구분? 섹션 구분?). page-1·page-2 둘 다 4번째 위치가 active로 표시되어 있어 시안상 active 위치가 페이지별로 갱신 안 된 것으로 보임 → **active 인덱스는 현재 페이지(여기선 4)에 동적으로 바인딩**해야 함. 사각 마커 의미는 기획 확인 필요.
- **㎖ 글자**: 단일 문자 "㎖"(U+33E1 아님, 흔히 "㎖" U+339C 계열) 사용. 폰트 미지원 시 "mL"로 폴백 처리 또는 "ml" 텍스트 조합 고려.
- **화살표·로고 한자 폰트**: ‹ › 글리프는 Pretendard 의존. 폰트 로드 전 FOUT 대비. "手"는 Noto Serif KR 로드 필요.
- **가격 카드 우측 정렬 정밀도**: 가격 숫자는 우측 정렬(`text-align:right`)이고 라벨은 좌측 정렬. 카드 폭 변동 시에도 라벨↔가격 양끝 정렬(`space-between`) 유지.
- **회전 스티커의 클리핑**: 캔버스 라운드(14px) + `overflow` 처리 시 스티커가 캔버스 가장자리에 걸치면 잘릴 수 있음. 스티커 레이어를 캔버스보다 약간 넓은 별도 오버레이로 두거나 `overflow:visible` 검토.
- **터치 타깃**: ‹ › 화살표와 dot은 시각적으로 작음(글리프/3.5px). 손님 태블릿 터치를 위해 실제 탭 영역(hit area)을 44px 이상으로 확장 권장.
