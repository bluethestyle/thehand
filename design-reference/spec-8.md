# spec-8 — 관리④ 메뉴 항목 관리 (판매중/일시품절/닫힘 + 보관함)

## 1. 목적/역할
사장(관리자)이 메뉴 항목별로 **판매 상태(판매중 / 일시품절 / 닫힘)** 를 토글하고, 닫은(숨긴) 항목을 **보관함**에서 한 번에 되살리는 관리자 전용 화면. "품절은 손님 화면에 표시만, 닫힘은 손님 화면에서 숨김, 닫아도 데이터는 보관함에 보존" 이라는 운영 규칙을 구현한다.

세로 태블릿(768 기준, 시안 캔버스 높이 812). 위→아래 단일 컬럼 흐름, 가로 풀폭. 좌우 본문 거터는 약 20px(컨테이너 패딩). 영역은 위에서 아래로 5개 블록:

1. **상단 앱바(헤더)** — 높이 64. 다크(`#111418`) 풀폭 바. 좌측 브랜드 로고+타이틀, 우측 액션 2개.
2. **안내 배너(서브헤더)** — 헤더 바로 아래, 높이 약 52. 크림색(`#FBF1DC`) 풀폭, 하단에 얇은 골드 구분선. 2줄 운영 안내 카피.
3. **"판매 중" 섹션** — 섹션 헤더(좌측 타이틀 + 우측 카운트) + 사케 행 카드 4장의 세로 리스트. 각 카드 우측에 3-state 상태 세그먼트 토글.
4. **구분선** + **"닫힌 메뉴 · 보관함" 섹션 헤더**(타이틀 + 개수 뱃지 + 우측 "손님 화면 비노출" 메타) + 보관 항목 카드 3장(반투명, 우측 "다시 판매하기" pill 버튼).
5. **하단 정보 스트립(푸터)** — 연녹색(`#F0F5F1`) 라운드 박스, 중앙 정렬 1줄. 보존 안내 카피.

전체 배경은 `#F4EFE6`(본문), `<body>`는 `#E7E2D7`. 실제 구현에서는 본문이 viewport 높이를 넘으면 세로 스크롤. 헤더는 sticky 권장(스펙상 상단 고정).

레이아웃 토큰(반응형 기준값):
- 컨테이너 좌우 패딩: 20px (카드 폭 = 100% - 40)
- 카드 좌측 내부 패딩: 첫 요소(썸네일) x=36 → 내부 16px
- 본문 카드 간 세로 간격: 약 10px (카드 높이 70 + gap 10 = 80 step)

## 5. 추론한 실제 인터랙션 (라이브 앱 동작)

1. **상태 세그먼트 토글(3-state)**: 각 판매중 카드의 "판매중 / 일시품절 / 닫힘" 세그먼트는 단일 선택 토글.
   - "판매중" 선택 → 손님 화면 정상 노출, 활성 알약 녹색(`#1F8A5B`).
   - "일시품절" 선택 → 손님 화면에 "일시 품절"로 **표시만**(주문 비활성/회색 처리), 활성 알약 적색(`#C84B31`) + 상품명 옆 인라인 뱃지 등장.
   - "닫힘" 선택 → 카드가 판매중 리스트에서 제거되어 **보관함 섹션으로 이동**, 손님 화면에서 숨김. 보관함 개수 뱃지 +1, "닫은 날" = 오늘 날짜로 기록.
2. **다시 판매하기 버튼**: 보관함 카드의 pill 탭 → 해당 항목을 판매중 리스트로 복원(상태 "판매중"), 보관함에서 제거, 개수 뱃지 -1. 가격·설명·산지·사진 등 메타 그대로 복원(재입력 없음).
3. **완료 버튼**: 변경사항 저장/적용 후 관리자 모드 종료(또는 이전 관리 화면으로 복귀).
4. **더보기(⋯)**: 추가 액션 메뉴(정렬, 일괄 닫기, 보관함 비우기 등) 오버플로 메뉴.
5. **상품명 옆 메타/카드 본문 탭**(추정): 항목 상세/편집 화면 진입 가능(시안에 직접 단서는 없음 → 옵션).
6. **리스트 스크롤**: 항목이 많아지면(판매중 8종, 보관함 3) 세로 스크롤. 헤더/배너는 고정 권장.
7. 상태 변경 시 마이크로 트랜지션: 활성 알약이 세그먼트 사이를 슬라이드(약 150~200ms), 색 전환.

## 6. 구현 주의점 · 모호한 부분

- **개수 뱃지 텍스트 색 누락**: 시안의 "3" 뱃지 `fill="undefined"` → 다크 텍스트(`#2D3436` 권장)로 명시 지정 필요.
- **상태색 토큰**: 판매중 = `#1F8A5B`(녹색), 일시품절 = `#C84B31`(적벽돌), 닫힘 = 활성 알약 색 미정(시안에 닫힘 활성 케이스 없음). 구현 시 닫힘 활성 알약 색을 별도 정의 필요(예 회색 `#6B6258` 또는 적색). 비활성 라벨은 모두 `#9A8F80`.
- **세그먼트 폭**: 트랙 210, 알약 66 고정이지만 세그먼트 3개를 균등 분배하려면 `flex: 1`로 구현하고 활성 알약을 active index에 맞춰 translate 하는 방식 권장(시안의 절대좌표 66/70 혼재는 무시).
- **보관함 카드 opacity vs 버튼**: 카드 그룹은 0.62 투명, 다시 판매하기 버튼은 불투명. CSS로는 카드 콘텐츠에만 opacity를 주고 버튼은 그 바깥(또는 `opacity:1` 재설정)에 두어야 시안과 동일.
- **썸네일 placeholder 두 종류**: 판매중 = 점선 테두리(`#D8D0C4` dashed) 위 크림 채움 / 보관함 = 점선 없는 솔리드 `#E4DDD0`. 상태에 따른 시각 차이 유지.
