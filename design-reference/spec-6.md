# spec-6 · 관리② 페이지당 항목 수 ↔ 글자크기(자동 비례) 비교

## 1) 목적 / 역할

관리자 모드 안에서 **"한 페이지(페이지형 메뉴판 한 장)에 사케를 몇 종 담을지"** 를 슬라이더로 정하고, 그 값에 따라 **메뉴 글자 크기가 자동 비례로 줄거나 커지는 결과**를 좌(4종·큰 글씨) / 우(8종·작은 글씨) 두 개의 라이브 프리뷰로 나란히 비교시켜 주는 설정 화면.

---

## 2) 전체 레이아웃 구조

세로 768 폭 태블릿 기준. 위에서 아래로 4개 스택 영역이 흐른다. (SVG 캔버스 높이 694는 이 화면 콘텐츠 높이일 뿐, 실제 페이지는 1024 높이 안에서 본문 영역이 늘어나도 무방.)

1. **상단 글로벌 헤더 (다크 바)** — 높이 64, 폭 풀블리드. 좌측 브랜드 로고+모드 라벨, 우측 액션 버튼 2개.
2. **섹션 인트로 바 (베이지)** — 헤더 바로 아래 높이 약 52, 풀블리드. 섹션 제목 + 보조 설명 1줄. 하단에 얇은 구분선.
3. **컨트롤 패널 카드 (흰 카드)** — 본문 좌우 여백 20 안에 폭 풀(728) 카드. 슬라이더 2개(페이지당 항목 수 / 글자 크기 미세조정)를 세로로 쌓음.
4. **비교 프리뷰 영역** — 가운데 정렬된 캡션 1줄 + 그 아래 **2열 그리드로 좌·우 미니 메뉴판 카드**(각 폭 300, 정사각 300). 두 카드 사이 가운데에 양방향 관계를 설명하는 **세로 2줄 화살표 라벨**(◀ 항목↓ 글씨↑ / 항목↑ 글씨↓ ▶)이 떠 있음.

정렬 원칙:
- 본문 좌우 거터: 헤더/인트로는 풀블리드, 컨트롤 카드는 좌우 20, 프리뷰 두 카드는 좌 24 / 우 24 안쪽으로 두 카드를 양 끝 정렬(중앙 약 120 갭).
- 모든 카드는 흰 배경 + 카드 드롭섀도(아래 토큰), 라운드 10~14.
- 반응형: 좁은 폭에서는 프리뷰 2열 → 1열 세로 스택, 가운데 화살표 라벨은 두 카드 사이 가로 배치로 회전 가능.

## 5) 실제 인터랙션 (정적 시안에서 추론)

1. **페이지당 항목 수 슬라이더** — 드래그/탭/키보드 화살표로 2~10 정수값 조정. 값이 바뀌면:
   - 우측 값 라벨 "N 종" 갱신.
   - 두 프리뷰가 아니라(시안은 4·8 고정 예시), 실제 앱에서는 선택값 N에 따라 **미리보기 행 수와 폰트가 실시간 재계산**(글자크기 = base × f(N), N↑ → 글자↓).
   - 데이터 총 종수 > N 이면 "현재 M종 → ⌈M/N⌉페이지로 자동 분할" 캡션 갱신, 페이지 dot 개수도 갱신.
2. **글자 크기 미세조정 슬라이더** — 자동 비례값을 기준으로 −/+ 미세 오프셋(예: −2~+2 단계). 값 라벨 "자동 ±n" 갱신, 프리뷰 폰트에 가산 반영.
3. **완료 ✓ 버튼** — 설정 저장 후 관리자 패널/이전 화면으로 복귀.
4. **⋯ (오버플로)** — 추가 옵션 메뉴(초기화/도움말 등) 팝오버.
5. **프리뷰 카드** — 좌=낮은 밀도/큰 글씨, 우=높은 밀도/작은 글씨의 양 극단 예시. 라이브에서는 슬라이더 값에 연동되어 동기 변형(좌우는 데모용 고정 비교일 수도, 또는 "현재값 vs 한 단계 차이" 비교일 수도 — 6) 모호점 참고).
6. **하단 dot 인디케이터** — 손님 화면에서의 페이지 스와이프를 미리 보여주는 표시(설정 화면에서는 비인터랙티브 미리보기로 두는 것을 권장).

## 6) 구현 주의점 · 모호한 부분

- **자동 비례 공식 미명시**: 시안 수치(4종→14.72px, 8종→9.6px / 가격 11.5→7.5 / 썸네일 폭 14.8→9.7)에서 역산하면 대략 **글자크기 ≈ 상수 / N** 또는 **선형 보간**으로 근사 가능. 정확 공식은 PM/디자인 확인 필요. (참고: 14.72 ≈ 58.9/4, 9.6 ≈ 76.8/8 — 단순 1/N은 아니고 완만한 감쇠 곡선. clamp(min,max) 적용 권장.)
- **좌·우 카드의 관계 모호**: (a) 단순 4종 vs 8종 데모 고정인지, (b) "현재 슬라이더 값 결과 vs 비교 기준" 인지, (c) 둘 다 슬라이더에 연동되는지 불명확. 우선은 "동일 컴포넌트 + N만 다른 두 인스턴스"로 구현하고 데이터는 props로 주입.
- **슬라이더 값과 프리뷰 불일치**: 슬라이더 값은 "8 종"인데 좌측 프리뷰는 4종. 좌측은 비교용 반대 극단 예시일 가능성. UI 카피로 "예시" 명시 필요.
- **카테고리 밴드 위쪽만 라운드**: SVG가 26px 라운드바 + 10px 직각바를 겹쳐 표현. 실제로는 `border-radius: 10px 10px 0 0` 한 요소로 구현.
- **썸네일 dashed placeholder**: 보더 `dashed 2 2`는 CSS `border: 0.8px dashed #D8D0C4` + `background:#F5F1EC` 로 매핑. 비율(폭:높이 ≈ 1:2.4) 유지.
- **가격 표기 단위 없음**: 숫자만(예: 18,500). 통화기호/단위는 디자인 의도대로 생략하되, 실제 메뉴에서 "원" 노출 여부 확인 필요.
- **dot 인디케이터 개수**: 시안은 pill1+dot4(=5)로 그려져 컨트롤 캡션의 "2페이지 자동 분할"과 숫자 불일치. 실제로는 ⌈총종수 / N⌉ 로 동적 산출해야 함(시안은 일러스트 목적의 임의 개수).
- **폰트 로딩**: 사케명은 Noto Serif KR(세리프), 그 외 UI는 Pretendard(산세리프). 두 폰트 모두 한글/한자 글리프 포함 웹폰트 사전 로드 필요.
- **반응형**: 프리뷰 2열은 좁은 폭에서 1열로 접고, 가운데 화살표 라벨은 카드 사이 가로 배너로 전환. 슬라이더 트랙은 컨테이너 폭에 맞춰 fluid.
- **letter-spacing**: "관리자 모드" 라벨만 0.08em, 나머지는 normal.
