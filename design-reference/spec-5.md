# spec-5 — 관리 ① 페이지 보드 (순서 / 삽입 / 숨김)

> 원본 시안: `design-reference/page-5.html` (768×1138 SVG, 태블릿 768×1024 세로 기준 / 본문은 세로 스크롤로 확장)

---

## 1. 목적 / 역할 (한 줄)

관리자가 디지털 메뉴판의 **전체 페이지(11장)를 한 화면에서 보고, 끌어서 순서를 바꾸고, 사이에 새 이미지·이벤트 페이지를 삽입하고, 손님 화면에서 숨길지 결정**하는 페이지 관리 보드.

---

## 2. 전체 레이아웃 구조

세로 한 줄 흐름. 위에서 아래로 3개의 띠(band) + 스크롤되는 리스트.

```
┌─────────────────────────────────────────────────────┐
│  [A] 글로벌 헤더 (다크, 고정 height 64)               │  ← sticky top
├─────────────────────────────────────────────────────┤
│  [B] 서브 헤더 / 툴바 (크림색, 56)                    │  ← sticky 가능
│      좌: 안내 문구 2줄   /   우: "+ 페이지 삽입" 버튼  │
├─────────────────────────────────────────────────────┤
│  [C] 페이지 카드 리스트 (세로 스택, 스크롤)            │
│      카드 11개, 카드 사이마다 "삽입선(+)" 구분자       │
│      각 카드: [핸들][번호][썸네일][제목+뱃지][우측 액션]│
└─────────────────────────────────────────────────────┘
```

- **레이아웃 모델**: 바깥은 `flex-direction: column`. 헤더/툴바는 고정 높이, 리스트는 `flex: 1; overflow-y: auto`.
- **헤더 [A]**: 좌측 로고 락업(골드 사각 + 手 + "관리자 모드"/"메뉴 페이지 관리" 2줄), 우측에 `완료 ✓` 주황 pill 버튼 + `⋯` 더보기 아웃라인 버튼. `justify-content: space-between`.
- **툴바 [B]**: 좌측에 안내 텍스트 2줄(`flex-start`), 우측에 주황색 큰 알약 버튼 1개. 하단 1px 골드 라인으로 본문과 구분.
- **리스트 [C]**: 카드는 좌우 풀폭(컨테이너 패딩 20px), 카드 높이 78px 고정. **카드 사이에 점선 + 가운데 `+` 원**으로 된 "여기에 삽입" 구분자가 끼어 있음(카드 N개 → 구분자 N개, 마지막 카드 뒤에는 없음 — 시안엔 1↔2, 2↔3 … 10↔11 사이 + 카드10↔11 위에 존재; 마지막 11 뒤에는 없음).
- 카드 내부는 좌→우 `flex`, 정렬 `align-items: center`:
  `[드래그핸들 ⠿] · [원형 번호] · [썸네일 46×62] · [제목블록(제목 + 타입뱃지 + 서브텍스트)] · spacer · [우측 액션 3버튼]`

## 3. 컴포넌트 사양

### 3.1 글로벌 헤더 (AdminTopBar)
- 배경 `#111418`, 높이 64px, 풀폭. 좌우 패딩 24px.
- **로고 마크**: 28×28, radius 7, 골드 그라데이션(`#f0c97a → #c0792a`, 좌상→우하). 안에 `手`(Noto Serif KR 700, 15px, `#1e1e36`).
- **로고 텍스트(2줄)**: 윗줄 "관리자 모드" — Pretendard 800, 9.5px, `#E7B85A`, letter-spacing 0.08em. 아랫줄 "메뉴 페이지 관리" — Pretendard 800, 14px, `#F3EDE2`.
- **완료 버튼**: 78×32, radius 16(완전 알약), 배경 `#F39C12`, 텍스트 "완료 ✓" Pretendard 800 12.5px `#1e1e36`.
- **더보기 버튼**: 36×32, radius 9, 투명 배경 + 보더 `#4A4D55` 1.3px, 안에 `⋯` 15px `#B8AE9C`.

### 3.2 서브 헤더 / 툴바 (PageBoardToolbar)
- 배경 `#FBF1DC`, 높이 56px, 하단 보더 `#F0E0BE` 1px.
- **안내 텍스트(좌, 2줄)**: 윗줄 "11개 페이지 · 끌어서 순서 변경" Pretendard 700 13px `#7A5A1E`. 아랫줄 "숨김 페이지는 손님 스와이프에서 빠집니다 (손님엔 10장 노출)" Pretendard 500 11.5px `#9A8154`.
  - 카운터 `11`과 노출 `10`은 데이터 바인딩(총 페이지수 / 숨김 제외 노출수).
- **삽입 버튼(우)**: 204×34, radius 17(알약), 배경 `#F39C12`, 텍스트 "+ 이미지·이벤트 페이지 삽입" Pretendard 800 12.5px `#1e1e36`.

### 3.3 페이지 카드 (PageCard) — 핵심 반복 컴포넌트
- 크기: 풀폭(컨테이너 좌우 20px 패딩 안), 높이 78px, radius 12, 배경 `#fff`.
- 그림자: `0 2px 5px rgba(26,26,46,0.12)` (SVG feDropShadow dy2/blur5 환산).
- 내부 좌측부터의 요소 위치(컨테이너 좌측 기준 대략):
  1. **드래그 핸들** `⠿` 16px `#D8D0C4` (Pretendard). 좌측 ~20px 지점. `cursor: grab`.
  2. **번호 배지**: 지름 26 원(r13), 배경 `#F5F1EC`, 가운데 숫자 Pretendard 800 13px. (SVG에 `fill="undefined"` 버그 — 실제 구현 색은 진한 텍스트색 `#2D3436` 또는 골드 `#C8902F` 권장. 아래 "주의점" 참고.)
  3. **썸네일** 46×62, radius 5, 흰 배경 + 보더 `#EDE8E0` 1px. 좌측 색 띠 또는 내부 미니어처가 카드 타입을 표현(§3.4).
  4. **제목 블록**: 메인 타이틀 Noto Serif KR 700 15.5px `#2D3436` + 우측에 **타입 뱃지**(§3.5) + 그 우측에 **서브텍스트** Pretendard 500 11.5px `#9A8F80`.
  5. **우측 액션 3버튼**(§3.6): 카드 우측 끝, 가로 배열.
- **상태 변형 — 숨김(disabled-like)**: 카드 전체 `opacity: 0.55`, 배경 `#F5F1EC`(흰색 대신), 그림자 없음. (10번 "음료(준비중)" 카드가 이 상태.)
- **상태 변형 — 드래그 중**(추론): 들어올린 카드 그림자 강조 + 약간 확대, 원래 자리에 플레이스홀더.

### 3.4 썸네일 미리보기 유형 (ThumbPreview) — 4종
썸네일(46×62 흰 박스) 안의 그림으로 페이지 종류를 시각화. **좌측 4px 컬러 띠**가 카테고리 색 코드:

| 유형 | 내부 그림 | 좌측 띠/포인트 색 | 등장 카드 |
|---|---|---|---|
| 표지(브랜드) | 검정 채움 + 골드 `手`(Noto Serif KR 700 18px `#E2A04F`) | 띠 없음 | 1 표지 |
| 텍스트 리스트(메뉴) | 회색 가짜 텍스트 라인 8줄(`#D8D0C4`/`#EDE8E0`) | 좌측 4px 띠 (색=카테고리) | 2·3·7·8·10·11 |
| 이미지 페이지 | 점선 액자(2 2 dash) + 산 실루엣 + 동그라미(해/태그) | 점선·산 색 = 테마색 | 4(초록 `#1F8A5B`) · 9(빨강 `#C0392B`) |
| 지도/플롯 | 십자축(`#EDE8E0`) + 컬러 점 3개 | 점 색 | 5 산지(`#2A6FAF`) · 6 취향(`#C45E86`) |

좌측 4px 띠 색 매핑(텍스트형 카드):
- 니혼슈 ①/② → `#C8902F` (골드)
- 쇼츄 → `#9B6FB0` (보라)
- 요리 → `#C0560F` (주황·갈)
- 음료(숨김) → `#9A8F80` (그레이)
- 원산지 표기 → `#636E72` (다크그레이)

### 3.6 우측 액션 버튼 그룹 (CardActions) — 카드당 3개
- 각 버튼: 지름 30 원(r15), 배경 `#F5F1EC`, 가로 간격 34px. 아이콘은 stroke 1.4px.
1. **미리보기(눈)**: 눈 모양 path + 중앙 점, stroke `#636E72`.
   - 숨김 카드(10)에서는 **눈-슬래시**(눈 + 대각선) + 색 `#9A8F80` (= "숨김됨" 토글 표현).
2. **편집(연필)**: 연필 path. (SVG `stroke="undefined"` 버그 → 실제 `#636E72` 권장.)
3. **순서 이동(▲▼)**: 위 `▲` / 아래 `▼` 두 화살표 세로 스택, Pretendard 800 9px `#9A8F80`. 드래그 대신 한 칸씩 위/아래 이동.

### 3.7 삽입 구분자 (InsertDivider) — 카드 사이

## 4. 화면에 박혀 있는 텍스트 / 데이터 (빠짐없이)

**헤더/툴바**
- "관리자 모드" / "메뉴 페이지 관리"
- "완료 ✓"  ·  "⋯"
- "11개 페이지 · 끌어서 순서 변경"

## 5. 추론한 실제 인터랙션 (라이브 앱 동작)

- **드래그 정렬**: 핸들 `⠿`을 잡고 카드를 위/아래로 드래그 → 페이지 순서 재배열. 드롭 시 번호 자동 재계산. (`고정` 뱃지 카드 1·11은 드래그 잠금 또는 양 끝 고정.)
- **▲▼ 버튼**: 드래그 대신 한 칸씩 위/아래 이동(접근성/정밀 이동용). 맨 위/아래에서 해당 방향 버튼 비활성.
- **삽입선 `+` 탭**: 두 카드 사이를 눌러 그 위치에 새 이미지/이벤트 페이지 생성(타입 선택 모달 추론).
- **툴바 "+ 이미지·이벤트 페이지 삽입"**: 리스트 끝(또는 선택 위치)에 새 페이지 추가하는 메인 진입점.
- **눈 아이콘 토글**: 손님 노출 ↔ 숨김 전환. 숨김 시 카드 opacity 0.55 + 눈-슬래시 + [숨김] 뱃지로 변하고 툴바의 "노출 N장" 카운터 감소. (10번이 그 상태 예시.)
- **연필 아이콘**: 해당 페이지 편집 화면으로 이동(메뉴 항목 관리 / 이미지 스티커 편집기 등 카드 타입별 라우팅).
- **카드 본문 탭(추론)**: 미리보기 또는 편집 진입.
- **완료 ✓**: 변경사항 저장 후 관리자 모드 종료/돌아가기.
- **⋯ 더보기**: 추가 옵션 메뉴(전체 초기화, 손님 화면 미리보기 등).
- **자동 분할 동작(데이터 규칙)**: "사케 8종"처럼 항목이 페이지 정원 초과 시 시스템이 자동으로 페이지를 쪼갬 — 관리자는 카테고리만 관리하고 실제 노출 페이지 수는 파생됨. 따라서 "11 관리 페이지"와 "손님 10장 노출"은 숨김 + 자동분할 결과로 달라질 수 있음.

## 6. 구현 주의점 · 모호한 부분

- **SVG의 `fill="undefined"` / `stroke="undefined"`**: 카드 번호 텍스트와 일부 연필 아이콘 stroke가 `undefined`로 비어 있음(원본 템플릿 버그). 구현 시 명시 색 지정 필요 — 번호는 `#2D3436`(또는 좌측 띠 카테고리색), 연필 stroke는 `#636E72` 권장.
- **viewBox 768×1138 vs 태블릿 1024**: 본문이 1024를 넘으므로 리스트는 세로 스크롤 영역으로 구현. 헤더/툴바 sticky 권장.
- **반응형**: SVG는 고정 좌표지만 목표는 반응형. 카드 폭은 컨테이너 100%(좌우 패딩 20px), 내부는 flex로 풀고 제목/서브텍스트는 좁은 폭에서 줄바꿈·말줄임 처리. 우측 액션 3버튼은 항상 우측 고정.
- **타입 뱃지 색 시스템**: 10% 틴트 배경은 `colorMix` 또는 8자리 hex(`#RRGGBB1A`) CSS 변수로 토큰화. 6종 타입(고정/자동 분할/이미지/탐색/이벤트/숨김)을 enum으로.
