# spec-1 — 손님① 니혼슈 (등급 카테고리 + 8종/페이지 자동분할 + dot 인디케이터)

> 원본 시안: `design-reference/page-1.html` (768×1024, 태블릿 세로). 좌표값은 비율/흐름/정렬로 환산해 기술. 모든 px 값은 768 폭 기준 디자인 단위이며 실제 구현은 반응형(컨테이너 폭에 비례)으로 옮긴다.

---

## 1) 목적 / 역할

손님용 디지털 메뉴판의 **니혼슈(일본 청주) 카탈로그 페이지**. 등급(정미보합) 카테고리 밴드로 묶어 사케를 나열하고, 한 페이지에 최대 8종을 자동 분할해 좌우 스와이프로 넘기는 **페이지형(paged) 뷰**. 각 사케는 판매단위 3종(잔술·도쿠리·보틀)의 ml·가격과 상태 뱃지를 보여 준다.

---

## 2) 전체 레이아웃 구조

세로 1열 풀폭 스택. 위에서 아래로 5개 영역:

1. **글로벌 헤더 (고정 바)** — 높이 60px, 다크(`#111418`) 풀폭 바. 좌측 브랜드 로고+워드마크, 우측 페이지 한자 태그(`日本酒`).
2. **페이지 타이틀 블록** — 높이 ~48px, 밝은 배경. 가운데 정렬 카테고리명(`니혼슈`) + 서브카피.
3. **본문 메뉴 리스트 (스크롤/페이지 콘텐츠)** — 좌우 패딩 28px. **카테고리 밴드 헤더**와 **사케 행(row)** 이 번갈아 나오는 세로 리스트. 각 행은 풀폭, 행 사이 1px 디바이더. 본문 좌우 바깥 가장자리에 ‹ › 페이지 화살표 오버레이(세로 중앙).
4. **푸터 (고정 바)** — 높이 78px, 밝은 배경, 상단 1px 디바이더. 좌측 "← 좌우로 넘기기" 힌트, 가운데 dot 인디케이터, 우측 "N / 10" 페이지 카운터.

내부 본문 그리드(사케 행 1개의 가로 구성):
- **컬럼 A — 보틀 썸네일** : 좌측 고정폭(42px, 행 높이 약 72px). 점선 placeholder.
- **컬럼 B — 텍스트 블록** : 가변폭(썸네일 우측 84px부터 시작). 제품명/산지/양조장/설명 4줄, 우측 끝에 인라인 부가 라벨(데움/한정/단위제한).
- **컬럼 C — 가격칩 3종** : 우측 고정 영역(잔술·도쿠리·보틀). 각 칩 폭 약 60px 간격으로 우정렬, 화면 우측 끝(740px)에 붙음.

> 카테고리 밴드는 컬럼 구분 없이 풀폭 1줄. 사케 행 높이는 88px 단위(밴드 26px + 간격)로 일정.

## 4) 컴포넌트 명세

### 4.1 글로벌 헤더 (`<AppHeader>`)
- 풀폭 다크 바, 배경 `#111418`, 높이 60px.
- **로고 마크**: 28×28, radius 7, 골드 선형 그라데이션(`#f0c97a → #c0792a`, 좌상→우하). 좌측 패딩 28px. 안에 한자 `手`(Noto Serif KR, 15px/700, `#1e1e36`, 중앙).
- **워드마크**: 로고 우측. `THE HAND` (Pretendard 14px/800, `#F3EDE2`, letter-spacing 0.14em). 그 아래 `사케 바 · 디지털 메뉴판` (Pretendard 10px/400, `#B8AE9C`).
- **우측 태그**: `日本酒` (Pretendard 12px/700, `#C9A24B`, letter-spacing 0.1em, 우정렬, 우측 패딩 28px).

### 4.2 페이지 타이틀 블록 (`<PageTitle>`)
- 밝은 배경, 가운데 정렬.
- 메인: `니혼슈` (Noto Serif KR 20px/700, `#2D3436`).
- 서브: `지역·등급으로 고르는 사케 · 모두 차게 제공` (Pretendard 10.5px/400, `#9A8F80`). 메인 바로 아래 붙음(행간 좁음).

### 4.3 카테고리 밴드 헤더 (`<CategoryBand>`)
- 풀폭(본문 패딩 28px 내부), 높이 26px, 배경 `#F5F1EC`, radius 6.
- 좌측: **한자 등급명** (Noto Serif KR 14px/700, `#2D3436`) + 그 우측에 **한글 등급명** (Pretendard 11.5px/700, `#C8902F`). 두 라벨은 인라인 좌정렬, 한자 폭만큼 띄움.
- 우측: **등급 설명** (Pretendard 10px/400, `#9A8F80`, 우정렬).
- 인스턴스 3종(데이터는 §5):
  - 純米大吟醸 / 준마이 다이긴죠 / `정미보합 50%↓ · 가장 화려`
  - 純米吟醸 / 준마이 긴죠 / `향과 맛의 균형`
  - 純米 / 준마이 / `쌀의 감칠맛`

### 4.4 사케 행 카드 (`<SakeRow>`)
풀폭, 높이 약 72px 콘텐츠 + 하단 1px 디바이더(`#EDE8E0`). 3 컬럼:

- **보틀 썸네일 placeholder** (좌, 42×72, radius 7): 배경 `#F5F1EC`, 1px **점선** 테두리(`#D8D0C4`, dasharray 3 3). 중앙 하단에 `보틀` (Pretendard 9px/400, `#B5AB99`). → 실제 보틀 이미지 자리.
- **텍스트 블록** (중앙):
  - 1행 **제품명** (Noto Serif KR 17.5px/700, `#2D3436`) — 추천 제품은 앞에 ★(`#E0962E`)와 제품명 위 **형광펜 하이라이트 바**(`rgba(243,156,18,0.42)`, 제품명 글자 뒤 반투명 밴드).
  - 제품명 우측 인라인 **(산지)** (Pretendard 12px/400, `#636E72`).
  - 제품명 우측에 인라인 **NEW** 뱃지(Pretendard 9px/800, `#C84B31`, 작은 superscript 위치)일 수 있음.
  - 2행 **양조장명** (Pretendard 11px/400, `#9A8F80`).
  - 3행 **설명** (Pretendard 12px/400, `#5C564E`).
  - 행 우측 끝(텍스트 영역 우상단 또는 우하단)에 **인라인 상태 라벨**(데움/여름 한정/도쿠리·잔술만)이 우정렬로 붙을 수 있음.
  - **일시 품절** 은 텍스트가 아니라 채워진 칩(아래 4.6)으로 제품명 옆에 표시.
- **가격칩 3종** (우, 4.5).

상태 변형:
- **추천(★+하이라이트)**: 첫 행(에미시키 몬순)에만 적용.
- **품절칩 동반**: 치요무스비 고우리키.
- **단위 제한**: 보틀 단위 비활성(opacity 0.4 + 가격 `—`) — 스이게이 코우이쿠(`도쿠리·잔술만`).

### 4.5 판매단위 가격칩 (`<PriceUnit>`) ×3
세로 스택(아이콘 → 단위명 → ml → 가격), 가운데 정렬, 칩 폭 ~28px·간격 약 60px, 우측에서 좌로 [잔술][도쿠리][보틀] 순. 행마다 동일 3칩.

- **아이콘** (라인 아이콘, 면 `#8A7F6E22` 반투명 + 스트로크 `#8A7F6E` 1.4):
  - 잔술 = 작은 와인잔/텀블러(12×14)
  - 도쿠리 = 도쿠리 병(14×17)
  - 보틀 = 720 보틀(12×22)
- **단위명** (Pretendard 9px/700, `#636E72`): `잔술` / `도쿠리` / `보틀`.
- **용량 ml** (Pretendard 7.5px/400, `#9A8F80`): `100㎖` / `300㎖` / `720㎖` (전각 ㎖ 사용).
- **가격** (Pretendard 12.5px/700, `#2D3436`): 천단위 콤마, 원화 기호 없음(숫자만). 예 `18,500`.
- **비활성 상태**: 그룹 opacity 0.4, 가격은 `—` 로 표시.

### 4.6 일시 품절 칩 (`<SoldOutBadge>`)
- 알약형, 폭 ~64px·높이 19px, radius 9.5, 배경 `#C84B31`.
- 텍스트 `일시 품절` (Pretendard 11px/800, `#fff`, 중앙).
- 위치: 해당 사케 제품명+산지 우측 인라인.

### 4.7 인라인 상태 라벨 (`<RowFlag>`, 텍스트 전용)
행 우측 끝 우정렬, Pretendard 10px/700:
- `♨ 데움 가능` → `#C0560F`
- `여름 한정` → `#1F8A8C`
- `도쿠리·잔술만` → `#8A7F6E`
- `NEW` → `#C84B31` (9px/800, 제품명 옆 인라인)

### 4.8 페이지 화살표 (`<PageArrow>` ‹ ›)
- 본문 좌우 바깥 가장자리, 세로 중앙(본문 높이 중앙 ~512px 라인).
- 글리프 ‹ / › (Pretendard 30px/300, `#D8D0C4`), 그룹 opacity 0.5(비강조).
- 좌/우 각 1개. 이전/다음 페이지 이동 트리거.

### 4.9 푸터 — dot 인디케이터 + 카운터 (`<Pager>`)
- 풀폭 78px 바, 상단 1px 디바이더.
- 좌측: `← 좌우로 넘기기` (Pretendard 11px/500, `#9A8F80`).
- 가운데: **dot 인디케이터** — 페이지 수만큼의 마커가 가로 일렬(중앙 정렬, 간격 ~20px).
  - 비활성 페이지: 작은 점 circle r=3.5, `#D8D0C4`.
  - **현재 페이지**: 가로로 늘어난 **pill** (18×7, radius 3.5, `#F39C12`).
  - 일부 마커는 작은 사각 아웃라인(7×7, radius 1.5, 1.4px 스트로크 `#D8D0C4`)으로 그려져 있음 → 시안상 혼합 표기. 구현 시 **모두 동일한 dot/pill 규칙으로 통일** 권장(§6 참고).
- 우측: `N / 10` 카운터 (Pretendard 12px/700, `#9A8F80`). 이 시안은 `2 / 10`.

## 6) 추론한 실제 인터랙션 (라이브 앱)

1. **좌우 스와이프 / 화살표 탭으로 페이지 넘기기**: 터치 스와이프(좌→우 = 이전, 우→좌 = 다음)와 ‹ › 탭이 동일하게 페이지 전환. 총 10페이지(`N / 10`), 현재 2페이지. 페이지 전환 시 dot 인디케이터 활성 위치 이동 + 카운터 갱신. 첫/마지막 페이지에서 화살표 비활성(현재 opacity 0.5는 상시 약하게 표현된 상태로 보임).
2. **8종 자동 분할 페이지네이션**: 데이터 소스의 사케 목록을 카테고리 순서로 정렬 후 **페이지당 최대 8행**으로 자동 분할(카테고리 밴드는 페이지 경계에서 다시 그려짐). 페이지 수는 데이터 길이에 따라 동적(`총 페이지 = ceil(...)`).
3. **상태 뱃지의 실데이터 연동**:
   - `일시 품절`: 재고 0 → 칩 표시. (선택) 해당 행 dim 처리 가능.
   - `NEW`: 신규 입고 플래그.
   - `여름 한정` / `♨ 데움 가능` / `도쿠리·잔술만`: 시즌·서빙·판매단위 속성 플래그.
   - `도쿠리·잔술만`: 보틀 단위 칩을 비활성(opacity 0.4, 가격 `—`)으로 렌더.
4. **추천 강조**: ★+형광펜 하이라이트는 "추천(featured)" 플래그가 켜진 사케에만 적용.
5. **정적 카탈로그(주문 기능 명시 없음)**: 시안엔 담기/수량 버튼이 없음 — 가격칩은 **표시 전용**으로 보는 것이 안전. (주문 연동은 미정 → §7.)
6. **반응형**: 768px 고정 시안이지만 태블릿/모바일 폭에 맞춰 본문 패딩·폰트·가격칩 영역이 비례 축소. 가격칩 3종이 좁은 폭에서 줄바꿈되지 않도록 우측 영역 최소폭 확보.

## 7) 구현 주의점 / 모호한 부분

- **dot 인디케이터 표기 불일치**: 시안의 마커가 점(circle)·사각 아웃라인(rect)·활성 pill·작은 채워진 사각이 섞여 있고 개수(약 9개)가 `10`과 맞지 않음. → 실제로는 **페이지 수(10)만큼 동일 dot**, 현재 페이지만 pill 로 통일 구현 권장. 사각 아웃라인은 SVG 작도 잔재로 간주.
- **활성 pill 색(`#F39C12`)** 과 인라인 라벨/뱃지 색 체계가 다름 → 토큰으로 분리 관리.
- **㎖ 전각 문자(`㎖`)** 사용 — 폰트에 글리프 없을 수 있으니 `ml` 대체 또는 폰트 보장 필요.
- **가격 표기**: 통화기호·"원" 없이 숫자+콤마만. 통화 단위 표기 정책 확인 필요(부가세/봉사료 각주 없음).
- **형광펜 하이라이트 바**: 제품명 글자 길이에 맞춰 폭이 정해짐(첫 행 110.6px). 구현은 텍스트 배경(`background: linear-gradient`/`box-shadow`)으로 글자 폭에 자동 맞춤되게.
- **NEW 뱃지 위치**: 제품명 baseline 위(superscript)로 그려져 있어 인라인 정렬이 까다로움 → 제품명과 baseline 정렬 또는 작은 상단 뱃지로 재해석.
- **카테고리-사케 그룹핑 로직**: "8종/페이지 자동분할"과 카테고리 밴드 재출력 규칙(페이지 분할 시 이어지는 카테고리의 밴드를 다음 페이지 상단에 다시 그릴지) 명세 필요.
- **헤더/푸터 고정 여부**: 시안상 콘텐츠와 분리된 바이므로 sticky/fixed 권장하나, 페이지형 뷰에서 스크롤이 없을 수 있어(한 화면 = 한 페이지) 레이아웃 정책 확정 필요.
- **주문 액션 부재**: 가격칩 탭 시 동작(상세/담기) 미정 — 표시 전용으로 시작.
- **색상 헥스는 SVG에서 직접 추출**했으므로 디자인 토큰과 1:1 매칭 가능. `#8A7F6E22`는 알파 포함 8자리 hex(13% 불투명).
