# spec-7 · 관리③ 이미지 페이지 스티커 편집기

> 출처 시안: `design-reference/page-7.html` (768×884 정적 SVG, 피그마 핸드오프)
> 대상: 태블릿 768×1024 세로 · Next.js + React + CSS Modules / CSS 변수 · 반응형 실제 DOM 재구현

---

## 1) 목적 / 역할

관리자가 "이미지형 메뉴 페이지"(사진 한 장으로 채워지는 페이지)의 **이미지 위에 뱃지·가격·가격 라벨 등 스티커를 끌어다 놓고 위치를 조절**하는 비주얼 편집기. 스티커 팔레트에서 캔버스로 드래그앤드롭, 배치된 스티커 선택·이동, 저장까지가 한 화면에서 끝난다.

---

## 2) 전체 레이아웃 구조

세로 스택, 3개 수평 밴드로 분할 (위→아래):

1. **상단 앱바 (관리자 헤더)** — 고정 높이, 다크. 좌측 브랜드/타이틀, 우측 액션(저장, 더보기).
2. **편집 캔버스 (본문)** — 헤더 아래 가장 큰 영역. 점선 테두리의 이미지 플레이스홀더 카드. 그 위에 배치된 스티커들(편집/선택 상태 포함)이 절대 위치로 떠 있음.
3. **스티커 팔레트 (푸터/도크)** — 화면 하단 고정 밴드, 연한 골드 배경. 안내 문구 1줄 + 드래그 가능한 스티커 칩 그리드(2행).

```
┌────────────────────────────────────────────┐
│ [手] 관리자 모드 / 이미지…스티커  [저장✓][⋯] │  앱바 (다크, h≈64)
├────────────────────────────────────────────┤
│  ┌── 점선 카드(이미지 placeholder) ──────┐  │
│  │   🟢계절 한정주(회전)      ⚪잔술만(회전)│  │  캔버스 (밝은 베이지 패널)
│  │        [산 아이콘 + 캡션]               │  │
│  │     ┌가격 스티커(선택/핸들)┐ [드래그…]  │  │
│  └────────────────────────────────────────┘  │
├────────────────────────────────────────────┤
│ 스티커 — 끌어다 …                            │  팔레트 (골드 도크)
│ [계절한정주][잔술만][일시품절][NEW][추천]… │  뱃지 칩 1행
│ [＄가격 스티커]  [+ 직접 입력]              │  가격칩 1행
└────────────────────────────────────────────┘
```

정렬 원칙(좌표 복제 금지, 흐름/그리드로):
- 앱바: `flex`, 좌측 브랜드 그룹 + `margin-left:auto`로 우측 액션 그룹 밀기.
- 캔버스: 패널(연베이지) 안에 점선 카드, 카드는 `aspect-ratio` 또는 `flex:1` 로 가용 높이 채움. 스티커 레이어는 카드 위 `position:absolute; inset:0`.
- 팔레트: 세로 스택(안내문 → 칩 행들). 칩 행은 `display:flex; flex-wrap:wrap; gap`.
- 스티커의 SVG `rotate(-6)` / `rotate(5)` 회전은 "손으로 붙인 느낌" 연출이므로 배치된 스티커에 `transform: rotate(deg)`로 유지(개별 데이터값). 팔레트 칩은 회전 없음.

## 3) 컴포넌트 카탈로그

공통 토큰은 §하단 "토큰" 표 참고. 색은 시안 hex 그대로.

### 3.1 AppBar (관리자 헤더)
- 배경 `#111418`, 높이 64px, 풀블리드(가로 100%).
- **브랜드 로고칩**: 28×28, `border-radius:7px`, 골드 그라데이션(`#f0c97a → #c0792a`, 좌상→우하). 중앙에 한자 「手」, Noto Serif KR 700, 15px, `#1e1e36`.
- **타이틀 블록**(로고 우측, 좌측정렬):
  - 상단 오버라인 「관리자 모드」 — Pretendard 800, 9.5px, `#E7B85A`, letter-spacing 0.08em.
  - 하단 타이틀 「이미지 페이지 편집 · 스티커」 — Pretendard 800, 14px, `#F3EDE2`.
- **저장 버튼**(우측): pill 78×32, `border-radius:16px`, 채움 `#F39C12`(주황). 라벨 「저장 ✓」 Pretendard 800, 12.5px, `#1e1e36`, 가운데정렬.
- **더보기 버튼**: 36×32, `border-radius:9px`, 투명 채움 + 1.3px 스트로크 `#4A4D55`. 글리프 「⋯」 15px `#B8AE9C`.

### 3.2 EditorCanvas (이미지 플레이스홀더 카드)
- **외곽 패널**: 연베이지 `#EDE7DD`, `border-radius:14px`, 캔버스 영역 전체 패딩 박스. (시안: x20 y80 w728 h620)
- **점선 드롭존 카드**(패널 안쪽): 채움 없음, 1.5px 스트로크 `#D8D0C4`, `stroke-dasharray:6 5` 점선, `border-radius:8px`. 패널 내부에 ~12px 인셋.
- **빈 상태 일러스트**(중앙, opacity 0.45): 산/사진 아이콘 — 80×60 라운드 사각 프레임 + 원(해) + 산 폴리라인, 모두 `#A99E8C` 2px 스트로크.
- **빈 상태 캡션**: 「쿠보타 만쥬 한정주 이미지」 Pretendard 700, 12.5px, `#8E8576`, 중앙정렬. (실제 앱에선 업로드된 이미지의 파일명/설명 자리)
- 이미지가 있으면 점선/일러스트/캡션 대신 실제 `<img object-fit:cover>`로 대체.

### 3.3 배치된 뱃지 스티커 (PlacedBadgeSticker) — 캔버스 위
캔버스에 이미 붙어 있는 2종(상태 변형 예시):
- **(A) 솔리드 깃발형** 「계절 한정주」: 본체 사각(≈97×28, `border-radius:4px`) + 우측 끝 삼각 꼬리(깃발/리본 모양). 채움 `#1F8A5B`(녹색). 텍스트 흰색 Pretendard 800, 13px. `rotate(-6deg)`. 강한 드롭섀도(`dy3 blur4 #1a1a2e@0.28`).
- **(B) 아웃라인 pill형** 「잔술만 가능」: 92.5×27 pill `border-radius:13.5`, 흰 채움 + 1.5px 스트로크 `#D8D0C4`. 텍스트 `#2D3436` Pretendard 800, 12.5px. `rotate(5deg)`. 동일 드롭섀도.
- 두 형태는 §3.5 팔레트 칩의 "배치된 인스턴스"이며, 회전·그림자만 추가됨.

### 3.4 가격 스티커 (선택/편집 상태) — PriceSticker (Selected)
캔버스 위에 **현재 선택된** 스티커. 여러 부속으로 구성:
- **본체 카드**: 흰색 `#fff`, 170×44, `border-radius:10px`, 드롭섀도.
  - 1행 단위/용량 「잔술 100㎖」 Pretendard 500, 12px, `#636E72`, 중앙.
  - 2행 가격 「24,000원」 Pretendard 800, 15px, `#2D3436`, 중앙.
- **선택 테두리(셀렉션 박스)**: 본체보다 살짝 큰 182×56 사각, 채움 없음, 1.6px 스트로크 `#F39C12`(주황), `stroke-dasharray:5 4` 점선, `border-radius:12px`.
- **4모서리 리사이즈 핸들**: 각 코너에 원 r5.5, 흰 채움 + 2px 스트로크 `#F39C12`.
- **회전 핸들**: 셀렉션 상단 중앙에서 위로 뻗는 1.6px 주황 선 + 끝에 원(r7, 흰 채움/주황 스트로크) 안에 「↻」 글리프(Pretendard 700, 9px, `#F39C12`).
- **드래그 힌트 툴팁**: 셀렉션 우측 하단에 다크 pill 118×26 `border-radius:13`, 채움 `#111418`. 텍스트 「드래그로 위치 이동」 Pretendard 700, 10.5px, `#F3EDE2`.
- 상태: 비선택 시 본체 카드만 보임(테두리/핸들/툴팁 숨김). 선택 시 위 부속 전부 노출.

### 3.5 스티커 팔레트 도크 (StickerPalette) — 푸터
- **밴드 배경** `#FBF1DC`(연골드), 상단 경계선 1px `#F0E0BE`. 풀블리드.
- **섹션 안내문**: 「스티커 — 끌어다 이미지 위 원하는 위치에 붙이세요」 Pretendard 700, 13px, `#7A5A1E`, 좌측정렬.
- **뱃지 칩(1행, 드래그 소스)** — 모두 높이 32, `border-radius:16px`(pill), 텍스트 Pretendard 800, 12.5px, 중앙정렬. 칩 폭은 라벨에 맞춰 가변(시안 폭은 참고치). 그림자: `card`(`dy2 blur5 @0.12`). 좌→우:
  1. 「계절 한정주」 — 채움 `#1F8A5B`, 텍스트 `#fff`. (≈96.5w)
  2. 「잔술만 가능」 — 흰 채움 + 1.4px `#D8D0C4` 스트로크, 텍스트 `#2D3436`. (≈96.5w)
  3. 「일시 품절」 — 채움 `#C84B31`(벽돌빨강), 텍스트 `#fff`. (≈84w)
  4. 「NEW」 — 채움 `#F39C12`(주황), 텍스트 `#1e1e36`. (≈53w)
  5. 「추천」 — 채움 `#FBF1DC`(연골드), 텍스트 `#A9761C`(골드브라운). (≈55w)
  6. 「한정 수량」 — 흰 채움 + 1.4px `#D8D0C4` 스트로크, 텍스트 `#2D3436`. (≈84w)
- **가격 칩(2행, 드래그/액션 소스)** — 높이 34, `border-radius:10px`(사각라운드):
  - 「＄ 가격 스티커」 — 흰 채움 + 1.4px `#D8D0C4` 스트로크, 텍스트 Pretendard 800, 12.5px, `#2D3436`, 좌측정렬, 좌패딩 ~14px. (150w)
  - 「+ 직접 입력」 — 채움 없음, 1.4px `#D8D0C4` **점선**(`dasharray:4 3`) 스트로크, 텍스트 Pretendard 700, 12.5px, `#9A8F80`, 중앙. (140w) → "비어있는/추가" 상태 칩.

> 팔레트 칩 우상단에 `text-anchor:end`인 빈 캡션 슬롯(9px `#9A8F80`)이 마크업에 존재 — 사용 횟수/카테고리 등 작은 메타 라벨 자리로 예약된 것으로 보임(현재 텍스트 비어있음).

- 「手」(로고)
- 「관리자 모드」
- 「이미지 페이지 편집 · 스티커」
- 「저장 ✓」
- 「⋯」

**캔버스**
- 빈 상태 캡션: 「쿠보타 만쥬 한정주 이미지」
- 배치 뱃지: 「계절 한정주」, 「잔술만 가능」
- 가격 스티커 본체: 「잔술 100㎖」 / 「24,000원」
- 회전 핸들 글리프: 「↻」
- 드래그 힌트: 「드래그로 위치 이동」

**팔레트**
- 안내문: 「스티커 — 끌어다 이미지 위 원하는 위치에 붙이세요」
- 뱃지 칩: 「계절 한정주」 · 「잔술만 가능」 · 「일시 품절」 · 「NEW」 · 「추천」 · 「한정 수량」
- 가격 칩: 「＄ 가격 스티커」 · 「+ 직접 입력」

> 판매단위/용량 표기 규약: 잔술 단위 + ㎖(전각 ml, U+339C) + "원". 가격은 천단위 콤마(예: 24,000원).
> 이 페이지에는 도쿠리/보틀 가격칩, 한자 등급명, dot 인디케이터·‹ › 화살표·"N / 10" 카운터·취향지도 플롯 등은 **없음**(다른 페이지 소관). 본 스펙은 page-7에 실제로 존재하는 요소만 기술.

## 5) 추론한 실제 인터랙션 (라이브 앱 동작)

**팔레트 → 캔버스 (생성)**
- 팔레트 칩을 캔버스 위로 **드래그앤드롭** → 드롭 지점에 해당 스티커 인스턴스 생성. (HTML5 DnD 또는 pointer 기반 커스텀 드래그; 태블릿 터치 우선이므로 `pointerdown/move/up` + `touch-action:none` 권장.)
- 「＄ 가격 스티커」 드롭 → 가격/단위가 채워진 가격 스티커 생성(메뉴 데이터와 연동되거나 직전 선택 항목 값).
- 「+ 직접 입력」 → 빈 텍스트 스티커 추가 후 인라인 입력 모드 진입(직접 문구/숫자 입력).

**배치된 스티커 조작**
- **탭/클릭** → 선택. 선택 시 §3.4의 점선 셀렉션 박스 + 4코너 핸들 + 회전 핸들 + "드래그로 위치 이동" 툴팁 노출.
- **드래그** → 캔버스 내 위치 이동(절대좌표 갱신). 캔버스 경계로 클램프 권장.
- **코너 핸들 드래그** → 크기 조절(스케일). 비율 유지 여부는 정책 필요(§6).
- **회전 핸들(↻) 드래그** → 회전각 변경(시안의 -6°/5° 처럼 자유 회전).
- **빈 영역 탭** → 선택 해제(부속 UI 숨김).
- 삭제 동작(휴지통/길게누르기)은 시안에 명시 없음 → 추가 정의 필요(§6).

**캔버스/이미지**
- 빈 점선 카드 **탭** → 이미지 업로드/선택(파일 picker 또는 미디어 라이브러리). 드롭존이므로 이미지 파일 **드래그 드롭 업로드**도 지원 가정.

**헤더**
- 「저장 ✓」 → 현재 스티커 배치(좌표·회전·스케일·내용) 영속화 후 편집기 닫기/토스트.
- 「⋯」 → 추가 메뉴(되돌리기/초기화/삭제/도움말 등) 팝오버.

## 6) 구현 주의점 · 모호한 부분

**좌표 / 반응형**
- 시안 viewBox는 768×884로 1024 세로 중 일부만 그림. 실제는 768×1024 기준 캔버스를 `flex:1`로 늘려 빈 하단을 캔버스가 흡수. 픽셀 좌표 하드코딩 금지 — 스티커 위치는 **캔버스 대비 백분율(%)** 로 저장해 리사이즈/회전 대응.
- 스티커 위치 모델: `{ x%, y%, rotation(deg), scale }` 권장. 픽셀 저장 시 다른 해상도에서 어긋남.

**드래그 구현**
- 태블릿 터치 환경 → HTML5 네이티브 DnD는 터치 지원이 약함. **Pointer Events 기반 커스텀 드래그** 권장(`setPointerCapture`, `touch-action:none`). 드래그 중 스크롤/줌 방지.
- 팔레트→캔버스 드롭 시 드롭 좌표를 캔버스 로컬 좌표로 변환(`getBoundingClientRect`).

**상태/데이터 모호점**
- 가격 스티커의 값(「잔술 100㎖ / 24,000원」)이 **어떤 메뉴 항목과 바인딩**되는지 불명확 — 자유 텍스트인지, 메뉴 DB 참조인지 정책 필요.
- 「+ 직접 입력」으로 만든 스티커의 스타일(색/형태 선택 UI) 정의 없음.
- 팔레트 칩 우상단 빈 캡션 슬롯의 용도 미정(사용 횟수? 카테고리?) — 비워두거나 제거 결정 필요.
