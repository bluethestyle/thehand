# spec-3 — 손님③ 이벤트 이미지 페이지

> 출처 시안: `design-reference/page-3.html` (768×1024 세로, 태블릿 페이지형 디지털 메뉴판 / 피그마 핸드오프 정적 SVG)
> 페이지 식별: 10페이지 슬라이드 중 9번째 (`9 / 10`)

---

## 1. 목적 / 역할

손님용 메뉴판에서 **수입사·매장이 제공한 이벤트(10주년 등) 프로모션 이미지를 한 화면 가득 보여주는 "이미지 전용" 슬라이드**. 가격표가 아니라, 큰 비주얼 한 장 + 그 위에 자유 배치된 스티커/뱃지로 이벤트를 홍보하는 페이지다. (page-2 "수입사 한정주 이미지 페이지"와 동일한 템플릿 계열.)

---

## 2. 전체 레이아웃 구조

세로 1024 화면을 위→아래 3영역으로 분할. 가로는 100% 폭 기준 좌우 동일 여백(시안 28px → 토큰화).

```
┌──────────────────────────────────────────┐
│ HEADER (고정 높이, dark bar)               │  height 60
├──────────────────────────────────────────┤
│                                            │
│  STAGE (이미지 캔버스 = 점선 프레임)        │  flex: 1
│   - 중앙: 이미지 placeholder(아이콘+안내)   │
│   - 그 위에 자유 배치 스티커 레이어(absolute)│
│   - 좌/우 가장자리: ‹ › 페이지 화살표       │
│                                            │
├──────────────────────────────────────────┤
│ FOOTER (페이지 인디케이터 + 각주)           │  height 78
└──────────────────────────────────────────┘
```

- **HEADER**: 가로 flex. 좌측 = 로고 그룹(골드 그라데이션 사각 로고 + "手" + "THE HAND" + 서브타이틀), 우측 끝 = 섹션 카테고리 라벨("이벤트"). `justify-content: space-between`, 수직 중앙 정렬.
- **STAGE(본문)**: 헤더와 푸터 사이 전체를 채우는 `position: relative` 캔버스. 안쪽에 점선 라운드 보더 프레임이 있고, 프레임 정중앙에 이미지 placeholder(아이콘 + 2줄 안내문). 스티커들은 이 STAGE를 기준으로 `position: absolute` + 각자 `rotate(...)`로 살짝 기울여 붙는 자유 배치 레이어. 좌우 가장자리에 세로 중앙 정렬된 ‹ › 화살표.
- **FOOTER**: 상단에 얇은 구분선. 가운데 = 페이지 dot 인디케이터(가로 정렬), 좌측 끝 = "← 좌우로 넘기기" 안내, 우측 끝 = "9 / 10" 카운터.

> 핵심: STAGE는 "한 장의 이미지를 꽉 채우는 캔버스"이고, 스티커는 그 위에 떠 있는 오버레이다. 실제 구현 시 이미지가 들어오면 placeholder는 사라지고, 스티커 좌표/회전은 이미지와 독립적인 오버레이 좌표계로 유지한다.

## 4. 컴포넌트 명세

### 4.1 헤더 (`HeaderBar`)
- 높이 60, 배경 `--header-bg`, 풀폭.
- **로고 마크**: 28×28 사각, radius 7, 골드 그라데이션(좌상 `#f0c97a` → 우하 `#c0792a`). 중앙에 "手"(Noto Serif KR, 15px/700, `#1e1e36`). 좌측 여백 28px.
- **워드마크**: 로고 우측. 1행 "THE HAND"(Pretendard 14px/800, `#F3EDE2`, letter-spacing 0.14em), 2행 "사케 바 · 디지털 메뉴판"(Pretendard 10px/400, `#B8AE9C`). 좌측 정렬, 두 줄 세로 묶음.
- **카테고리 라벨**: 우측 끝, "이벤트"(Pretendard 12px/700, `#C9A24B`, letter-spacing 0.1em), 우측 여백 28px. → 페이지마다 바뀌는 슬롯(page-2는 "한정주").

### 4.2 스테이지 프레임 (`StageFrame`)
- 본문 영역 안쪽에 위치하는 점선 라운드 박스. 시안: x24 y78 w720 h856 → 좌우/상단 여백 24, 푸터 위까지.
- 채움 `#EDE7DD`, 보더 `1.5px` `#D8D0C4` **dashed (dash 6, gap 5)**, radius 14.
- 역할: "여기에 이미지가 들어온다"를 나타내는 드롭 영역. 이미지 업로드 시 이 프레임 안을 `object-fit: cover`로 채우고 점선/placeholder는 숨김.

### 4.3 이미지 Placeholder (`ImagePlaceholder`)
- 프레임 정중앙(가로 중앙, 세로 약 중앙)에 배치, **opacity 0.5** 그룹.
- 아이콘: 88×68 라운드(8) 빈 사각 보더(2px `#A99E8C`) + 좌상단 원(r8) + 산 모양 path(꺾은선). = "사진 없음" 픽토그램.
- 1행 텍스트: "수입사 제공 이미지 영역"(Pretendard 14px/700, `#8E8576`, 중앙).
- 2행 텍스트: "관리 화면에서 한정주·이벤트 이미지를 올립니다"(Pretendard 11.5px/400, `#A99E8C`, 중앙).
- 이미지가 실제로 있으면 전부 숨김.

### 4.4 스티커/뱃지 레이어 (`StickerLayer` + `Sticker`)
STAGE 위 `position:absolute` 오버레이. 각 스티커는 개별 회전. 이 페이지에 박힌 스티커 4종 + 변형:

| # | 종류 | 형태 | 색 | 텍스트 | 폰트 | 회전 |
|---|---|---|---|---|---|---|
| 1 | **리본(깃발) 뱃지** | 좌측 라운드(4) 직사각 + 우측 삼각 꼬리 | bg `#C0392B`, text `#fff` | "10주년 이벤트" | Pretendard 13px/800 | -6° |
| 2 | **알약 뱃지(채움)** | pill(radius=h/2) | bg `#F39C12`, text `#1e1e36` | "6월 한정" | Pretendard 12.5px/800 | +5° |
| 3 | **메모 카드** | 흰 라운드(10) 카드, sh2 그림자 | bg `#fff` | 제목 "키핑 보너스" + 본문 "키핑 1병당 잔술 1잔 무료" | 제목 Noto Serif KR 15px/700 `#2D3436`, 본문 Pretendard 12px/500 `#636E72` | -2° |
| 4 | **알약 뱃지(아웃라인)** | pill, 흰 채움 + 보더 | bg `#fff`, border 1.5px `#D8D0C4`, text `#2D3436` | "기간 6/1 – 6/30" | Pretendard 12.5px/800 | -3° |

- 공통: 그림자(리본/알약 = sh, 카드 = sh2), 가로 패딩 약 16px(텍스트 좌우 균형), 세로 높이 27~28(뱃지) / 58(카드).
- **리본 꼬리**: 몸통 우변에서 14px 튀어나온 삼각형, 몸통과 같은 색.
- page-2 대비 변형 참고: 원형 "한정/30병" 도장, 컬러 다른 리본(`#1F8A5B`) 등 — 같은 `Sticker` 컴포넌트의 variant(ribbon / pill-solid / pill-outline / card / stamp)로 일반화 권장.

### 4.5 페이지 화살표 (`PageArrow` ‹ / ›)
- STAGE 좌/우 가장자리, 세로 중앙. 글리프 ‹ › (Pretendard 30px/300, `#D8D0C4`, **opacity 0.5**).
- 좌 = 이전 페이지, 우 = 다음 페이지. 시각적으로 매우 옅음(보조 힌트).

### 4.6 푸터 / 페이지 인디케이터 (`FooterNav`)
- 높이 78, 배경 `#FDFCFB`, 상단 1px 구분선 `#EDE8E0`(좌우 28 여백).
- **dot 인디케이터(중앙 정렬, 가로 한 줄)**: 총 10슬롯이 일정 간격(시안 20px 피치)으로 배열. 슬롯 타입 3가지:
  - 일반 dot: 원 r3.5, `#D8D0C4`.
  - **활성 페이지 pill**: 18×7 라운드 pill(3.5), `#F39C12`. → 이 페이지에서는 **9번째 슬롯**이 활성.
  - **섹션 구분 마커**: 7×7 라운드(1.5) 빈 사각 보더(1.4px `#D8D0C4`). → 이 페이지에서는 **4번째 슬롯**(page-2와 동일 위치)에 존재. = "이미지 페이지 묶음/섹션 경계" 표식으로 추정.
- **좌측 각주**: "← 좌우로 넘기기"(Pretendard 11px/500, `#9A8F80`), 좌측 정렬.
- **우측 카운터**: "9 / 10"(Pretendard 12px/700, `#9A8F80`), 우측 정렬.

## 6. 추론한 실제 인터랙션 (라이브 앱 동작)

1. **좌우 페이지 스와이프**: 손님이 화면을 좌/우로 스와이프하거나 ‹ › 화살표를 탭하면 이전/다음 메뉴 페이지로 이동. 푸터 dot/카운터/헤더 카테고리 라벨이 함께 갱신. 총 10페이지 캐러셀. (각주 "← 좌우로 넘기기"가 이 동작을 안내.)
2. **dot 탭 점프**: 푸터 dot/pill을 탭하면 해당 페이지로 바로 점프(선택적). 활성 pill이 현재 페이지를 표시.
3. **이미지 표시**: STAGE는 본질적으로 "관리자가 올린 한 장의 이미지를 풀-블리드로 보여주는" 뷰. 이미지가 없으면 점선 프레임 + placeholder, 있으면 `object-fit: cover` 이미지로 대체.
4. **스티커는 정적 오버레이**: 손님 화면에서는 비인터랙티브(장식/홍보). 위치·회전·텍스트는 데이터로 주입.
5. **관리자 측(추론)**: 이 페이지는 관리 화면에서 (a) 이벤트 이미지 업로드, (b) 스티커 추가/삭제 + 텍스트·색·위치·회전 편집을 할 수 있어야 함(placeholder 문구 "관리 화면에서 … 올립니다"가 근거). 손님 뷰에서는 읽기 전용.
6. **무인 운영용 자동 슬라이드(추정)**: 디지털 메뉴판 특성상 일정 시간 무조작 시 자동으로 다음 페이지로 넘어가는 오토플레이가 있을 수 있음(미확정 — 6번 참조).

## 7. 구현 주의점 / 모호한 부분

- **반응형 매핑**: 시안은 768×1024 절대좌표. 실제 구현은 절대좌표 복제 금지 → 헤더(고정 60) / 푸터(고정 78) / 본문(flex:1) 의 flex 컬럼 레이아웃으로, 좌우 여백은 토큰(24~28px)으로. 스티커는 STAGE 기준 **백분율(%) 좌표 + rotate**로 배치해 다양한 화면비에서도 상대 위치를 유지.
- **스티커 회전과 클리핑**: 각 스티커가 ±몇 도 기울어 있어, 회전 후 STAGE 밖으로 삐져나가지 않도록 가장자리 여유와 `overflow` 정책을 정해야 함. 리본 삼각 꼬리는 SVG/`clip-path`로 구현(CSS border-triangle보다 안정적).
- **이미지 종횡비**: placeholder 영역은 4:3 비슷(88×68)이나 실제 이미지 비율은 미정. STAGE를 채울지(cover) 맞출지(contain), 그리고 cover 시 스티커가 이미지 위에 겹치는 가독성(어두운 사진 대비)을 고려.
- **dot 인디케이터의 "사각 마커" 의미 모호**: 4번째 슬롯의 빈 사각(7×7)은 일반 dot과 구분되는 특수 슬롯. page-2와 동일 위치에 있어 "이미지/이벤트 섹션 그룹 구분자"로 해석했으나, 단순 디자인 변주일 수도 있음 → 데이터 모델에서 `slotType: 'dot' | 'active' | 'section'` 로 표현하고 확정 필요.
- **활성 인디케이터 폭**: 활성 슬롯은 원이 아니라 18×7 가로 pill이라, dot 줄의 총 폭과 중앙 정렬 계산이 슬롯마다 달라짐. 정렬은 flex + gap으로 하고 활성 pill만 폭을 키우는 방식 권장.
- **폰트 로드**: Pretendard, Noto Serif KR 웹폰트 사전 로드 필요(특히 "手" 세리프, 카드 제목). 미로드 시 폴백 어긋남.
