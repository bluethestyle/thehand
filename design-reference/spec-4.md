# spec-4 — 손님 ④ 취향 지도 (달콤↔깔끔 · 화려↔은은)

> 원본 시안: `design-reference/page-4.html` (768×1024 세로, 태블릿 페이지형 디지털 메뉴판 1페이지)
> 페이지 라벨(주석): `손님 ④ 취향 지도 — 달콤↔깔끔·화려↔은은 (기존 탐색 페이지 유지)`
> 푸터 카운터: **6 / 10** · 페이지 인디케이터의 활성 pill 위치 = 6번째 (탐색 모드)

---

## 1) 목적 / 역할

손님이 **맛/향 취향을 2축 좌표(달콤↔깔끔 × 화려↔은은) 위에서 시각적으로 탐색**하고, 끌리는 위치의 **번호(1~8)** 를 아래 "번호로 찾기" 목록에서 사케 이름·산지와 매칭해 메뉴에서 찾도록 안내하는 **추천/탐색 보조 화면**.

---

## 2) 전체 레이아웃 구조

세로 1단 스택, 위→아래 4영역. 좌표가 아니라 흐름/정렬 기준으로 기술.

```
┌──────────────────────────────────────────┐
│ 글로벌 헤더 (고정 다크 바, h≈60)            │  ← 공통
├──────────────────────────────────────────┤
│ 페이지 타이틀 블록 (중앙 정렬, h≈58)         │
│   "취향 지도" + 보조설명 1줄                  │
├──────────────────────────────────────────┤
│ 본문                                       │
│   (A) 취향 지도 플롯 (정사각 카드, 중앙)      │
│       - 4사분면 + 십자축 + 축 라벨 4개        │
│       - 번호 점(node) 8개                    │
│   (B) "번호로 찾기" 범례 리스트              │
│       - 2열 그리드 (좌 1~4 / 우 5~8)         │
│   (C) 좌우 페이지 화살표 ‹ › (본문 세로중앙)  │
├──────────────────────────────────────────┤
│ 글로벌 푸터 (page 인디케이터 + 카운터, h≈78) │  ← 공통
└──────────────────────────────────────────┘
```

- **배경**: 페이지 전체 `#FDFCFB` (시안의 SVG body 배경 `#E7E2D7`은 뷰어 매트일 뿐, 페이지 캔버스 색은 `#FDFCFB`).
- **헤더/푸터는 1~8페이지 공통 컴포넌트** (page-3 등과 동일 마크업). 본문만 페이지별로 다름.
- 반응형: 768px는 기준 너비. 실제 DOM에서는 **태블릿 세로 고정폭(예: max-width 768, 좌우 safe-area)** 으로 잡되, 플롯은 정사각 비율(aspect-ratio 1/1) 유지, 범례는 2열→좁아지면 1열로 reflow.

### 4.1 글로벌 헤더 `<GlobalHeader accent="探索" />` (공통)
- 다크 바: 전폭 × 높이 60, 배경 `#111418`.

- 우측 끝 액센트 라벨: 본 페이지 = **"探索"**(12/700, `#C9A24B`, ls 0.1em, 우측 정렬). (페이지별로 문구 변경: page-3=이벤트 등)

### 4.2 페이지 타이틀 블록
- 배경 `#FDFCFB`, 전폭, 중앙 정렬.

### 4.3 취향 지도 플롯 `<TasteMapPlot />`
- **카드**: 정사각 (시안 420×420), `rx 18`, 배경 = 세로 그라데이션 `#F7F2E8`(0)→`#FDFBF7`(0.7), 외곽선 1px `#EDE8E0`. 본문 가로 중앙.
- **십자 축선**: 가로 1줄 + 세로 1줄, 색 `rgba(26,26,46,0.10)`, 카드 중앙 교차. 축선은 카드 내부 패딩(시안: 카드 174~594 폭에서 축선 216~552 = 약 10% 여백) 안쪽까지만.
- **내부 박스**: 축선 영역과 동일 범위의 사각 외곽선 `rx 10`, `none` 채움, stroke `rgba(26,26,46,0.07)` 1px (사분면 영역 시각 강조).
- **축 라벨 4개** (카드 바깥 또는 가장자리):
  - 상단 중앙: **"향이 화려 · 프레시"** (Pretendard 12/700, `#C45E86`).
  - 하단 중앙: **"향이 은은 · 안정적"** (Pretendard 12/700, `#8b6f47`).

  - 우측(세로, rotate +90): **"깔끔"** (Noto Serif KR 12/700, `#5dade2`).
  - 축 의미: **Y축 = 향(위=화려, 아래=은은)**, **X축 = 맛(좌=달콤, 우=깔끔)**.
- **노드 점 `<TasteNode>`** (8개): 원 r=15(플롯 안), 채움색은 데이터별, 중앙에 흰 숫자(12/800). 정규화 좌표(0~1 × 0~1)를 카드 내부 plot 영역에 매핑해 배치.

## 6) 추론한 실제 인터랙션 (라이브 앱 동작)

1. **페이지 좌우 스와이프 / 화살표 탭**: 태블릿 페이지형 메뉴판 — 손가락 가로 스와이프 또는 ‹ › 탭으로 1~10페이지 전환. 푸터 인디케이터·카운터(`6 / 10`)가 현재 페이지에 동기화(활성 pill 이동, 좌우 끝에서 화살표/스와이프 비활성).
2. **노드 점 ↔ 범례 행 연동(권장)**: 플롯의 번호 점과 "번호로 찾기" 행은 같은 데이터. 점 탭/호버 시 대응 범례 행 하이라이트(또는 그 반대). 상세(가격/판매단위 등)는 다른 페이지로 라우팅하거나 바텀시트로 노출 — 단 본 시안 자체엔 상세 트리거 UI 없음(탐색 보조 페이지).
3. **품절 상태(7번)**: 노드는 윤곽형(흐림), 범례는 칩 윤곽형 + "품절" 태그 → 선택 불가/비활성 처리. 데이터의 `soldOut: true`로 구동.
4. **취향 지도 자체는 정적 좌표 플롯**(드래그/슬라이더 입력 아님): 손님이 직접 좌표를 찍는 인터랙티브가 아니라, 메뉴들이 미리 배치된 "맵"을 읽고 번호를 찾는 read-only 시각화. (향후 확장으로 "내 취향 위치 핀 드래그→가까운 사케 추천"은 가능하나 시안엔 없음.)
5. **자동 슬라이드(키오스크 모드)**: 디지털 메뉴판 특성상 무입력 시 일정 시간 후 자동 페이지 순환 가능성(요구 시 옵션). 시안에 명시 없음.

## 7) 구현 주의점 / 모호한 부분

- **노드 좌표 매핑**: 시안 좌표는 viewBox 절대 px. 반응형 DOM에선 각 사케에 **정규화 좌표(x,y ∈ 0~1)** 를 부여하고 plot 내부영역(패딩 10%)에 `left/top %`로 절대배치하는 방식 권장. 시안 plot box = 216~552(폭 336)이므로 `nx = (cx-216)/336`, `ny = (cy-182)/336`로 역산.
- **노드 색 ↔ 사분면 불일치**: 색은 사분면 위치와 1:1이 아님(예: 7번은 윤곽형). 색은 데이터 속성으로 명시 부여할 것(자동 계산 금지).
- **축 방향 정의**: 시안 라벨 배치로 보아 **세로축 위=화려/아래=은은, 가로축 좌=달콤/우=깔끔**. 단 좌측 "달콤"·우측 "깔끔" 라벨이 X축 양끝에 있으므로 X=맛, Y=향이 자연스러우나, 한 차례 디자인팀 확인 권장.
- **푸터 인디케이터 슬롯 패턴**: dot/네모/pill 혼합. 네모(□)가 "이미지 전용/특수 페이지", pill이 "현재 페이지"로 추정되나 의미 명세가 시안에 없음 → 페이지 메타데이터(type)로 모델링하고 디자인 확인 필요. 본 페이지 활성 슬롯은 6번째.
- **헤더/푸터 공통화**: page-3과 헤더 구조·푸터 구조 동일. `<GlobalHeader>`/`<GlobalFooter>`로 추출하고 `accent`, `current/total`, 슬롯 패턴만 props로.
- **세로 텍스트 회전**: "달콤/깔끔"은 rotate ±90. CSS `writing-mode: vertical-rl` 또는 `transform: rotate()` 중 폰트 렌더링(Noto Serif KR) 품질이 나은 쪽 선택.
- **폰트 폴백**: Pretendard·Noto Serif KR 웹폰트 로드 전 레이아웃 시프트 방지(`font-display: swap` + size-adjust). 한자/한글 글리프 모두 필요.
- **"6 / 10"·활성 pill·페이지 라벨(探索)** 은 라우터 상태에서 파생되어야 하며 하드코딩 금지(다른 페이지와 컴포넌트 공유).
- 노드 r=15(플롯) vs 범례 칩 r=14로 미세하게 다름 — 의도된 차이이므로 분리 토큰으로 관리.
